Ветки в проекте
main

Стабильная версия проекта.
Только проверенный код, который показываем преподавателю.
Непосредственно в main никто не коммитит, изменения попадают сюда через merge из develop.
develop

Общая ветка разработки.
Сюда мержатся все feature/*-ветки.
Код может быть неидеальным, но должен хотя бы собираться и запускаться.
feature/*

Ветки для разработки конкретных задач/функционала.
Примеры:
feature/auth — регистрация/вход.
feature/supervisor — функционал супервайзора.
feature/landing — лендинг.
